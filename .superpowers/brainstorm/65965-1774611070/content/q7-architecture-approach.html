A

🟢 SPA + Supabase + Vercel Serverless

현재 보일러플레이트(Vite+React) 그대로 활용 + Supabase DB/Auth + Vercel API Routes

장점

  • 기존 프로젝트 구조 100% 활용
  • Supabase 무료 플랜으로 시작
  • Vercel 무료 배포 + 서버리스 API
  • mTLS 인증서는 Vercel에서 처리

단점

  • Cold start 지연 가능
  • 복잡한 실시간 기능에 한계

⭐ 추천: 비용 0원 시작, 검증된 앱인토스 패턴

B

🔵 SPA + bkend.ai BaaS

bkend.ai MCP로 DB/Auth/Storage 전부 처리. 서버 코드 최소화

장점

  • 서버 코드 거의 불필요
  • MCP 도구로 빠른 DB 설계
  • Auth/RBAC 내장

단점

  • 토스 OAuth mTLS 처리에 별도 서버 필요
  • 프로모션 API 호출에 커스텀 서버 필요
  • bkend.ai 플랫폼 의존
C

🟣 Next.js 풀스택

Next.js App Router로 마이그레이션. SSR + API Routes + Supabase

장점

  • SSR로 SEO + 초기 로딩 최적화
  • API Routes 내장 (별도 서버 불필요)
  • 풀스택 단일 프로젝트

단점

  • 기존 Vite 프로젝트 마이그레이션 필요
  • 앱인토스 granite.config 호환 확인 필요
  • 과도한 구조일 수 있음 (미니앱에 SSR 불필요)
