🔐 데이터 저장 & 로그인

사용자 데이터를 어디에 저장하고, 로그인은 어떻게 할까요?

A

📱 로컬 전용 (서버 없음)

토스 SDK Storage API로 기기 내 저장. 로그인 없이 바로 사용

  • ✅ 서버 비용 0원, 개발 빠름
  • ❌ 기기 변경 시 데이터 소멸, 랭킹 불가
  • ❌ 보상형 광고/토스포인트 지급에 로그인 필요할 수 있음
B

☁️ 서버 연동 + 토스 로그인 (추천)

토스 OAuth 로그인 → Supabase/Vercel 서버에 데이터 저장

  • ✅ 랭킹, 크로스 디바이스, 프로모션 리워드 가능
  • ✅ 인트로 화면 후 로그인 (심사 규칙 준수)
  • ❌ 서버 구축 필요 (Vercel 무료 플랜으로 시작 가능)
C

🔀 하이브리드 (게스트 + 로그인)

처음엔 로컬 저장(게스트)으로 사용 → 랭킹/리워드 원하면 토스 로그인

  • ✅ 진입 허들 최저, 필요할 때 로그인
  • ✅ 게스트 → 로그인 전환 시 데이터 마이그레이션
  • ❌ 두 가지 상태 관리 복잡도 증가
